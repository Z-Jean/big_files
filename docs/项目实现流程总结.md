## 一、项目概述

### 1.1 项目背景

本项目来源于全栈开发考题，围绕企业级大文件上传场景设计。在企业内部，经常需要上传 GB 级别的视频文件、安装包、大数据集等，普通上传容易因网络中断导致上传失败、需要重头开始。因此需要实现**分片上传 + 断点续传 + 秒传**机制。

| 数据库 | MySQL | 8.0 |
| 容器化 | Docker + Docker Compose | - |
| CI/CD | GitHub Actions | - |

### 1.3 评分标准

```
用户选择文件
    ↓
计算文件 MD5（spark-md5）
    ↓
调用秒传检测接口
    ↓
┌─────────────────────────────────────┐
│ 文件已存在？                         │
│ 是 → 显示"秒传成功"，结束            │
│ 否 → 继续                           │
└─────────────────────────────────────┘
    ↓

    ↓
┌─────────────────────────────────────┐

    ↓
┌─────────────────────────────────────┐
│ 用户点击暂停？                        │
│ 是 → 停止上传，保存进度               │
│ 否 → 继续上传下一个分片               │
└─────────────────────────────────────┘
    ↓
所有分片上传完成
    ↓
调用合并接口
    ↓
显示上传成功，刷新文件列表

def create_access_token(user_id: int) -> str:
    expire = datetime.utcnow() + timedelta(hours=settings.JWT_EXPIRE_HOURS)
    payload = {"sub": str(user_id), "exp": expire}
    return jwt.encode(payload, settings.JWT_SECRET, algorithm=settings.JWT_ALGORITHM)

def verify_token(token: str) -> int:
    payload = jwt.decode(token, settings.JWT_SECRET, algorithms=[settings.JWT_ALGORITHM])
    user_id = payload.get("sub")
    return int(user_id)
```

#### 4.3.2 MD5 计算（Web Worker）

**避免阻塞 UI**（`frontend/public/md5-worker.js`）：
```javascript
self.onmessage = function(e) {
    const { file } = e.data;
    const spark = new SparkMD5.ArrayBuffer();
    const reader = new FileReader();
    const chunkSize = 2 * 1024 * 1024; // 2MB
    let currentChunk = 0;

    const loadNext = () => {
        const start = currentChunk * chunkSize;
        const end = Math.min(start + chunkSize, file.size);
        reader.readAsArrayBuffer(file.slice(start, end));
    };

    reader.onload = function(e) {
        spark.append(e.target.result);
        currentChunk++;
        if (currentChunk < totalChunks) {
            loadNext();
        } else {
            self.postMessage({ type: 'complete', md5: spark.end() });
        }
    };

    loadNext();
};
```

#### 4.3.5 分片合并

**后端服务**（`backend/services/upload_service.py`）：
```python
async def merge_chunks(md5: str, filename: str, total_chunks: int, user_id: int, db: Session):
    # 生成唯一文件名
    file_ext = os.path.splitext(filename)[1]
    unique_filename = f"{hashlib.md5(md5.encode()).hexdigest()}{file_ext}"
    file_path = os.path.join(settings.UPLOAD_DIR, unique_filename)

    # 按顺序合并分片
    with open(file_path, "wb") as f:
        for i in range(total_chunks):
            chunk_path = os.path.join(settings.CHUNK_DIR, md5, f"{i}.chunk")
            with open(chunk_path, "rb") as chunk:
                f.write(chunk.read())

    # 保存文件记录到数据库
    file_record = File(
        md5=md5,
        original_filename=filename,
        file_path=file_path,
        file_size=os.path.getsize(file_path),
        user_id=user_id
    )
    db.add(file_record)
    db.commit()

    # 清理临时分片
    shutil.rmtree(os.path.join(settings.CHUNK_DIR, md5), ignore_errors=True)

    return file_record
```

#### 4.3.6 断点续传

**查询已上传分片**（`backend/routes/upload.py`）：
```python
@router.get("/chunks/{md5}")
async def get_uploaded_chunks(md5: str, db: Session = Depends(get_db)):
    chunks = db.query(Chunk).filter(
        Chunk.file_md5 == md5,
        Chunk.status == "completed"
    ).all()

    # 检查分片文件是否实际存在
    uploaded_indices = []
    for chunk in chunks:
        chunk_path = os.path.join(settings.CHUNK_DIR, md5, f"{chunk.chunk_index}.chunk")
        if os.path.exists(chunk_path):
            uploaded_indices.append(chunk.chunk_index)
        else:
            db.delete(chunk)

    db.commit()
    return {"uploaded_chunks": uploaded_indices}
```

### 5.1 本地开发阶段问题

#### 问题 1：uv sync 失败 - pyproject.toml 配置错误

**原因分析**：pyproject.toml 中缺少 hatch 构建配置

[tool.hatch.build.targets.wheel]

#### 问题 2：MySQL 连接失败 - 密码不匹配

**错误信息**：
```
pymysql.err.OperationalError: (1045, "Access denied for user 'root'@'localhost'")
```

**原因分析**：本地 MySQL 密码是 `root`，配置文件中写的是 `123456`

**解决方案**：修改 `backend/config.py` 中的密码配置

---

#### 问题 3：bcrypt 版本不兼容

**错误信息**：
```
AttributeError: module 'bcrypt' has no attribute '__about__'
```

**原因分析**：passlib 1.7.4 与 bcrypt 5.0.0 不兼容

**解决方案**：在 `requirements.txt` 中固定版本 `bcrypt==4.0.1`

---

#### 问题 4：JWT 令牌验证失败

**错误信息**：
```
JWTClaimsError Subject must be a string.
```

**原因分析**：jose 库要求 `sub` 字段必须是字符串，但代码传的是整数

**解决方案**：
```python
# 修改前
payload = {"sub": user_id, "exp": expire}  # user_id 是 int

# 修改后
payload = {"sub": str(user_id), "exp": expire}  # 转换为字符串
```

---

#### 问题 5：暂停上传报错

**错误信息**：
```
signal is aborted without reason
```

**原因分析**：点击暂停时 abort 请求被当作错误处理

**解决方案**：在 catch 块中检测 AbortError 并忽略
```typescript
if (err instanceof DOMException && err.name === 'AbortError') {
    return; // 这是用户主动暂停，不是错误
}
```

---

#### 问题 6：继续上传无效

**问题描述**：点击继续后进度条不动

**原因分析**：`isPaused` 是 React state，在 `startUpload` 函数的闭包中捕获的值不会更新

**解决方案**：使用 `useRef` 存储暂停状态
```typescript
const isPausedRef = useRef(false);

// 暂停时
isPausedRef.current = true;

// 继续时
isPausedRef.current = false;

// 上传循环中检查
if (isPausedRef.current) {
    return; // 实时检查最新状态
}
```

---

#### 问题 7：文件下载 404

**问题描述**：点击下载跳转到 404 页面

**原因分析**：前端 Next.js 只代理了 `/api/` 路径，没有代理 `/uploads/` 静态文件路径

**解决方案**：在 `next.config.js` 中添加代理规则
```javascript
async rewrites() {
    return [
        { source: '/api/:path*', destination: 'http://localhost:8000/api/:path*' },
        { source: '/uploads/:path*', destination: 'http://localhost:8000/uploads/:path*' },
    ];
}
```

---

#### 问题 8：下载需认证 - Not authenticated

**问题描述**：点击下载跳转页面显示 "Not authenticated"

**原因分析**：下载接口需要 JWT 认证，但 `<a>` 标签不会自动携带 token

**解决方案**：使用 JavaScript 调用下载接口，手动携带 Authorization header
```typescript
const handleDownload = async (fileId: number, filename: string) => {
    const token = localStorage.getItem('token');
    const response = await fetch(`/api/files/${fileId}/download`, {
        headers: { 'Authorization': `Bearer ${token}` }
    });
    const blob = await response.blob();
    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = filename;
    a.click();
};
```

---

#### 问题 9：下载文件名是随机的

**问题描述**：下载的文件名是 MD5 哈希，不是原始文件名

**原因分析**：后端为了防止文件名冲突，使用了 MD5 哈希作为存储文件名

**解决方案**：添加下载接口，使用 `FileResponse` 设置原始文件名
```python
return FileResponse(
    path=file.file_path,
    filename=file.original_filename,
    media_type='application/octet-stream'
)
```

#### 问题 10：删除文件后再次上传失败

**错误信息**：
```
FileNotFoundError: [Errno 2] No such file or directory: 'chunks\\xxx\\0.chunk'
```

**原因分析**：删除文件后，数据库中的分片记录还在，但实际分片文件已被清理

**解决方案**：查询已上传分片时，检查文件是否实际存在
```python
for chunk in chunks:
    chunk_path = os.path.join(settings.CHUNK_DIR, md5, f"{chunk.chunk_index}.chunk")
    if os.path.exists(chunk_path):
        uploaded_indices.append(chunk.chunk_index)
    else:
        db.delete(chunk)  # 删除无效记录
```

### 5.2 部署阶段问题

#### 问题 11：SSH 主机密钥验证失败

**错误信息**：

Host key verification failed

**原因分析**：服务器第一次连接 GitHub 时，GitHub 的主机密钥没有添加到 `known_hosts` 文件中

**解决方案**：在部署脚本中添加

git clone https://github.com/Z-Jean/big_files.git

#### 问题 13：Docker 构建超时

Run Command Timeout

#### 问题 14：MySQL 连接失败 - 后端启动时 MySQL 未就绪

**错误信息**：
```
Can't connect to MySQL server on 'mysql' ([Errno 111] Connection refused)

**原因分析**：MySQL 容器启动需要时间，但后端在 MySQL 还没准备好时就尝试连接

**解决方案**：修改部署脚本，先启动 MySQL，等待 60 秒

```

#### 问题 15：登录密码错误 - bcrypt 哈希格式不正确

**错误信息**：
```
passlib.exc.UnknownHashError: hash could not be identified
```

**原因分析**：通过 MySQL 命令行插入的 bcrypt 哈希值格式不正确，passlib 无法识别

**解决方案**：让后端应用自己创建用户，在 `main.py` 启动时自动创建
```python
def create_default_user():
    db = SessionLocal()
    try:
        user = db.query(User).filter(User.username == "admin").first()
        if not user:
            user = User(
                username="admin",
                password_hash=pwd_context.hash("123456")
            )
            db.add(user)
            db.commit()
    finally:
        db.close()

create_default_user()
```
