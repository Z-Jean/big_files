**二、大文件分片上传 + 断点续传（核心功能，45 分）**

**1. 业务场景描述**

企业内部经常需要上传 GB 级别的视频文件、安装包、大数据集等，普通上传容易因网络中断导致上传失败、需要重头开始。需要实现**分片上传 + 断点续传 + 秒传**机制：

• 将大文件切割成固定大小的分片（如 5MB / 片），逐个上传

• 网络中断后重新上传时，跳过已上传成功的分片

• 相同文件（MD5 相同）实现秒传，无需重复上传

核心需求：支持最大 2GB 单文件上传，分片上传、断点续传、秒传、实时进度显示、暂停 / 继续上传。

**前端（Next.js）功能**

• **上传页面**：/upload 页面包含：文件选择区域、上传按钮、暂停 / 继续按钮、取消按钮、总体进度条、每个文件的独立进度、上传状态提示、已上传文件列表

• **文件选择**：支持多选文件，支持所有格式，**单个文件最大支持 2GB**

• **文件预处理**：

• 计算文件 MD5（作为文件唯一标识，用于秒传和断点续传）

• 生成分片索引：第 0 片、第 1 片、第 2 片...

• **秒传检测**：上传前先调用后端接口，传入文件 MD5 和大小，若后端已存在该文件，直接返回 "上传成功"，实现秒传

• **分片上传**：

• 逐个上传分片，每个分片携带：文件 MD5、分片索引、总分片数、分片内容

• 支持**取消上传**，清除已上传分片

• **断点续传**：

• 上传前查询该文件已上传成功的分片列表

**数据库表设计**

• **chunks 表**：id, file_md5, chunk_index, status, created_at
